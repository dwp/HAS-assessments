User journeys Working assumptions

Task 1. Book TA assessment OR book F2F assessment
  • Appointment booker
  • Assumption - updates to status to split TA and F2F allows processes to scale and be itearted on with greater flexibility and outcomes as well as supporting stats, filtering and 'app booker' follow process decided by HCP at IR.

Task 2. Assign an HCP
  • Appointment booker
  • Assumption - there is an existing process to view 'assessment booked' and 'start assessment' as a list view for resource managers. Do they need to toggle between views as per HB tab design or could they simply open 2 browser windows if needed for MVP until we understand more?

Task 3. Monitors
  • Appointment booker
  • Assumption - App booker watches progress and status change (start assessment), no action until assessment complete or changes needed so assumption stays in caseload.
